Do-able Animation XML 1.0.0

Purpose

The purpose of the Do-able Animation XML format is to provide an easy-to-edit animation format for simple keyframed, 2d translation, animations.
The XML version used is 1.0, the encoding used is UTF-8.

Header

The header for XML 1.0 documents using utf-8 encoding is:

<?xml version="1.0" encoding="utf-8" ?>

Animations are defined in the "<animation>" block.
<animation>
<!-- Keyframes here -->
</animation>

Objects (animated textures) are defined in the "<objects>" block.
<objects>
<!-- Objects here -->
</objects>

Object

Objects are basically animated textures, the size of this texture is by default the size of the image resource.

Parameters

resource: The resource for the object, for example "data/melon.png".
parent: Parent alias (object follows parent), this will make position relative! (default=none).

The following parameters can be used in keyframes as well.
alias: The alias for the object, for example "melon"

x: The start position on the x-axis (basically keyframe at 0 seconds, default=0).
y: The start position on the y-axis (default=0).
r: The start rotation (default=0).

cr: The start red-value (default=255).
cg: The start green-value (default=255).
cb: The start blue-value (default=255).
ca: The start alpha-value (default=255).

These parameters can also be used for translations inside of a keyframe, also wrapped in an "<object>" block, which is inside an "<objects>" block.
<object>

<resource>content/pineapple_hd.png</resource>
<alias>papple</alias>
<x>%50</x>
<y>%50</y>

</object>

Keyframe

Keyframes are defined in seperate "<keyframe>" blocks.

Parameters

timestamp: This is when this keyframe occurs on the timeline, it must be equal or higher than 0 and can be a decimal.
interpolator: The interpolation to use (linear, cosine, cubic, default=linear).
Example keyframe:
<keyframe>

<timestamp>1.65</timestamp>
<interpolator>linear</interpolator>

</keyframe>
